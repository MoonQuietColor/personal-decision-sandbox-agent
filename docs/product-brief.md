# 产品说明：AI 个人决策沙盘 Agent

## 1. 项目解决的核心痛点

很多学生和独立开发者并不是没有目标，而是不知道目标是否现实、应该先做哪一步、怎样判断一个选择的机会成本。普通计划工具只会记录任务，AI 聊天工具容易给出空泛建议，二者都缺少“在行动前模拟结果”的能力。

本项目希望把个人目标转化为可推演对象：用户输入目标、截止时间、每天可投入时间、能力水平、可用资源和风险偏好后，系统生成多条路径、风险解释和阶段路线图，让用户在真正投入之前看到可能失败的原因与更稳妥的执行方案。

## 2. 核心逻辑流

```text
目标输入
  ↓
约束识别：时间 / 能力 / 资源 / 风险偏好
  ↓
复杂度估算与可行性评分
  ↓
多路径推演：保守 / MVP / 平衡 / 冲刺
  ↓
多 Agent 质询：规划 / 质疑 / 资源 / 执行 / 复盘
  ↓
输出路线图、风险雷达、导出文档
```

## 3. 多 Agent 分工

- **Planner Agent**：把目标拆成可执行路径。
- **Critic Agent**：主动寻找失败点，防止盲目乐观。
- **Resource Agent**：评估时间、工具、预算和算力是否够用。
- **Executor Agent**：把路径变成每日或阶段任务。
- **Reviewer Agent**：根据执行结果提出复盘与调整规则。

## 4. 与市面工具的差异

普通 To-do List 关注“做什么”，本项目关注“为什么这样做、是否做得完、失败点在哪里”。普通 AI 聊天助手强调回答，本项目强调可视化推演和执行闭环。

## 5. 当前版本边界

当前版本是零后端静态原型，重点展示产品逻辑、交互结构和可运行 Demo。推演规则基于本地启发式算法，不依赖真实大模型。这样做可以降低运行成本，也方便快速部署到 GitHub Pages。
